# 硬件设计与 PCB 打样规划

## 文档定位

本文档用于记录 Smart_Socket 硬件方案、模块接口、PCB 打样约束、设计决策和验证结果。

本文档当前只建立结构。后续内容必须经过逐节讨论后补充，不应把未确认的设想写成最终结论。

### 文档状态

| 项目 | 当前值 |
|------|--------|
| 文档状态 | 骨架已建立，内容待讨论 |
| 硬件版本 | 待定义 |
| 对应 PCB 版本 | 待定义 |
| 最近评审日期 | 待填写 |
| 当前打样阶段 | 方案整理 |

### 状态标记

| 标记 | 含义 |
|------|------|
| `初步设想` | 已提出方向，但尚未完成资料核对或方案评审 |
| `待确认` | 缺少参数、资料或决策 |
| `已确认` | 已根据资料和评审形成当前版本结论 |
| `已验证` | 已通过实物、仪器或测试流程验证 |
| `已废弃` | 曾经考虑，但不再用于当前版本 |

### 记录原则

1. 未经确认的器件参数不得作为 PCB 设计依据。
2. 关键参数应注明数据手册、原理图、实物测量或测试记录来源。
3. 市电侧设计、隔离方案和载流设计必须单独评审。
4. 设计变更应同步更新接口表、BOM、风险清单和验证计划。
